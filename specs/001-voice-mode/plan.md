# Implementation Plan: Conversational Voice Mode

**Branch**: `001-voice-mode` | **Date**: 2026-03-25 | **Spec**: `specs/001-voice-mode/spec.md`

## Summary

VS Code extension adding conversational voice input to Claude Code. A Python sidecar captures microphone audio, runs two-stage VAD (WebRTC → Silero ONNX), detects a wake word via openWakeWord, transcribes speech via faster-whisper, and delivers transcripts to the extension over a Unix domain socket. The extension bridges to Claude Code by simulating typing into its input field.

## Technical Context

**Language/Version**: TypeScript 5.x (extension), Python 3.11+ (sidecar)
**Primary Dependencies**: VS Code Extension API, faster-whisper, silero-vad (ONNX), openWakeWord, webrtcvad
**Storage**: Filesystem only — model cache at `~/.cache/claude-voice/models/`
**Testing**: Vitest (TypeScript), pytest (Python)
**Target Platform**: Linux (primary), macOS (stretch goal)
**Project Type**: VS Code extension + Python sidecar
**Constraints**: All processing local, all dependencies FOSS, <2s transcription latency for typical utterances

## Complexity Tracking

No constitution violations. All decisions follow the simplest viable approach.

| Area | Complexity Level | Justification |
|------|-----------------|---------------|
| Two-stage VAD | Moderate | Two-stage adds one dependency (webrtcvad) but is the proven pattern — single-stage Silero wastes CPU on silence |
| Unix socket IPC | Low | Slightly more complex than stdio but required for bidirectional async (user's explicit choice) |
| Command word detection | Low | Simple string matching on transcript suffix — no NLP needed |

## Phase 2: Python Sidecar

**Goal**: Working audio pipeline — mic capture → VAD → wake word → STT → transcript output over socket.

**Approach**: TDD for core logic. Fix-validate loop per phase.

### Tasks

**T008** — Implement error hierarchy (`sidecar/errors.py`) [spec: Infrastructure Decisions]
- `VoiceError` base with `code` and `message`
- Sidecar-side subclasses: `AudioError`, `TranscriptionError`, `DependencyError`
- Each error has a machine-readable code (e.g., `MIC_NOT_FOUND`, `MODEL_LOAD_FAILED`)
- Note: `SidecarError` and `IntegrationError` are extension-side (TypeScript) only — created in T026

**T009** — Implement protocol messages (`sidecar/protocol.py`) [FR-011, FR-014, FR-015, FR-016]
- Dataclasses for each message type: `StatusMessage`, `TranscriptMessage`, `ErrorMessage`, `ConfigMessage`, `ControlMessage`
- `serialize()` → JSON string + newline
- `deserialize(line)` → typed message object
- Unit tests: round-trip serialization for every message type

**T010** — Implement audio capture (`sidecar/audio.py`) [FR-022]
- `sounddevice` InputStream at 16kHz, mono, int16
- Configurable device selection (empty string = system default)
- Yields audio chunks (30ms frames for VAD compatibility)
- Error handling: `AudioError` on no mic, permission denied
- Unit tests: mock sounddevice, verify frame size and sample rate

**T011** — Implement two-stage VAD (`sidecar/vad.py`) [FR-020, FR-021]
- Stage 1: `webrtcvad.Vad(mode=3)` — aggressive mode, rejects obvious silence
- Stage 2: Silero VAD via ONNX Runtime — confirms speech on WebRTC-positive frames
- Pre-speech ring buffer (~300ms / 10 frames at 30ms) to avoid clipping
- Outputs: `speech_start`, `speech_end` events with buffered audio
- Configurable silence timeout (from settings, default 1500ms) for push-to-talk mode
- Unit tests with audio fixtures: silence → no events, speech → start/end events, pre-buffer preserves initial phonemes

**T012** — Implement wake word detection (`sidecar/wakeword.py`) [FR-030, FR-031, FR-032, FR-033]
- Load openWakeWord model (TFLite) from `models/` directory
- Process audio frames, emit wake word detection event
- Configurable wake word model name
- Strip wake word audio from the captured segment
- Unit tests: audio fixture with wake word → detected, without → not detected

**T013** — Implement command word detection (`sidecar/command_words.py`) [FR-050, FR-051, FR-052, FR-053, FR-054]
- After transcription, scan transcript suffix for submit/cancel words
- Strip matched command words from transcript
- Return action: `submit`, `cancel`, or `none` (no command word found — keep accumulating in continuous mode)
- Case-insensitive matching
- Unit tests: transcript "refactor this send it" → ("refactor this", submit), "do something never mind" → ("", cancel)

**T014** — Implement transcriber (`sidecar/transcriber.py`) [FR-040, FR-041]
- Load faster-whisper model by size name (tiny/base/small/medium)
- Model path: `~/.cache/claude-voice/models/`
- Transcribe audio buffer → text
- Error handling: `TranscriptionError` on model load failure, `DependencyError` if faster-whisper not installed
- Unit tests: audio fixture → expected transcript text (fuzzy match acceptable)

**T015** — Implement pipeline orchestration (`sidecar/pipeline.py`) [FR-060, FR-062, FR-063]
- Connects: audio → VAD → wake word gate → transcriber → command word detection
- Three modes: wake word (gate on), push-to-talk (gate off, external start/stop), continuous (gate off, command words delimit)
- Emits status events (speech_start, speech_end, processing) and transcript events
- Max utterance duration enforcement (default 60s)
- Unit/integration tests: feed audio fixture through pipeline, verify correct events and transcript

**T016** — Implement socket server (`sidecar/server.py`) [FR-010, FR-012, FR-013]
- Unix domain socket at `$XDG_RUNTIME_DIR/claude-voice-<pid>.sock` (fallback `/tmp`)
- Accept single connection (one extension client)
- Read NDJSON `config` and `control` messages from client
- Write NDJSON `status`, `transcript`, `error` messages to client
- Clean up socket file on shutdown
- Integration test: connect to socket, send config + start, read status messages

**T017** — Implement sidecar entry point (`sidecar/__main__.py`)
- Parse socket path from command line args
- Initialize pipeline with config from socket
- Signal handling (SIGTERM, SIGINT) for clean shutdown
- Logging to stderr at INFO level

## Phase 3: TypeScript Extension

**Goal**: Working VS Code extension — lifecycle management, socket client, status bar, Claude Code bridge.

### Tasks

**T018** — Implement protocol types (`src/protocol.ts`) [FR-011]
- TypeScript interfaces matching sidecar protocol (ConfigMessage, ControlMessage, StatusMessage, TranscriptMessage, ErrorMessage)
- Serialize/deserialize functions
- Unit tests: round-trip all message types, reject malformed JSON

**T019** — Implement socket client (`src/socket-client.ts`) [FR-010]
- Connect to Unix domain socket
- NDJSON line-buffered reader
- Send typed messages, receive typed messages via EventEmitter
- Reconnection on disconnect (for sidecar restart)
- Unit tests: mock socket, verify message framing

**T020** — Implement config adapter (`src/config.ts`) [FR-012]
- Read VS Code settings → build ConfigMessage
- Listen for settings changes → push updated config to sidecar
- Unit tests: settings object → correct ConfigMessage

**T021** — Implement status bar (`src/status-bar.ts`) [FR-001]
- State machine: Idle → Listening → Processing → Error
- Status bar item with icon and tooltip per state
- Click handler toggles listening (Idle ↔ Listening)
- Unit tests: state transitions, correct icon/tooltip per state

**T022** — Implement sidecar lifecycle manager (`src/sidecar.ts`) [FR-002, FR-003, FR-004, FR-005]
- Spawn Python process: `python -m sidecar --socket <path>`
- Monitor process exit, auto-restart with circuit breaker (3 crashes in 60s)
- Pass socket path to sidecar via command line
- Clean up socket file on stop
- Determine Python path (check `python3`, `python`, configurable)
- Unit tests: mock child_process, verify restart logic, circuit breaker

**T023** — Implement Claude Code bridge (`src/claude-bridge.ts`) [FR-070, FR-071, FR-072, FR-073, FR-074, FR-075]
- Open sidebar via `claude-vscode.sidebar.open` command
- Focus input via `claude-vscode.focus` command
- Simulate typing via `vscode.commands.executeCommand('type', { text })`
- Auto-submit: simulate Enter keypress after typing
- Paste-and-review: type text, leave cursor
- Queue for sequential delivery (no overlapping typing)
- Unit tests: mock VS Code API, verify command sequence

**T024** — Implement model manager (`src/model-manager.ts`) [FR-042, FR-043, FR-044, FR-090, FR-091]
- Check if model exists at `~/.cache/claude-voice/models/<model-name>/`
- Download from Hugging Face (faster-whisper model repo) with progress
- VS Code progress notification during download
- Clean up partial downloads on failure/cancel
- "Download Model" command: pick model size, trigger download
- Unit tests: mock fetch, verify progress reporting, partial cleanup

**T025** — Implement dependency checker (`src/commands.ts`) [FR-006, FR-092]
- "Check Dependencies" command: verify Python, faster-whisper, silero-vad, openwakeword, webrtcvad, sounddevice
- Run `python -c "import faster_whisper; ..."` to check each dependency
- Report results as VS Code notification (all good / list missing)
- Check for Claude Code extension via `vscode.extensions.getExtension('anthropics.claude-code')`
- On extension activation: check Claude Code extension, warn if missing

**T026** — Implement extension entry point (`src/extension.ts`)
- Register all commands
- Create status bar, sidecar manager, socket client, bridge
- Wire socket events: status → status bar, transcript → bridge
- Handle push-to-talk keybinding (register as VS Code keybinding)
- Dispose everything on deactivate
