# Learnings

Discoveries, gotchas, and decisions recorded by the implementation agent across runs.

---

- **T002**: VS Code `contributes` in package.json does not support declarative status bar items. Status bar items must be created programmatically via `vscode.window.createStatusBarItem()` in the extension code (T021). The task description mentions "status bar item" in contributes but this is not possible in the manifest.
- **T002**: Used `engines.vscode: "^1.85.0"` to match `@types/vscode` version — this is a reasonable baseline for 2024+ VS Code features.
- **T004**: `npm` and `node` are only available inside `nix develop` shell — all commands must be run via `nix develop --command bash -c "..."`.
- **T004**: Created `scripts/build.mjs` using esbuild JS API instead of inline CLI in package.json — cleaner and extensible. A minimal `src/extension.ts` stub is needed for typecheck and build to pass.
- **T005**: Audio fixture generator uses stdlib only (no numpy) for synthetic mode, so it works outside the nix devshell. The `--tts` mode requires piper-tts + numpy for realistic speech. Synthetic fixtures use modulated tone bursts that should activate VAD but won't produce meaningful transcriptions — downstream tests needing real transcription should use `--tts` regenerated fixtures.
- **T008**: `uv sync --dev` requires `[tool.uv] dev-dependencies` (or `[dependency-groups] dev`) — the `[project.optional-dependencies] dev` section alone is NOT sufficient. Without the uv-specific section, `uv sync --dev` actually *uninstalls* pytest. Added `[tool.uv] dev-dependencies` to pyproject.toml to fix.
- **T010**: `sounddevice` loads libportaudio.so.2 at import time, which fails in nix environments where the library path isn't set. Use lazy import (`_import_sounddevice()` helper) so the module can be imported and tested without the native library. Tests mock `_import_sounddevice` rather than `sidecar.audio.sd`.
- **T010**: For unit tests of callback-based audio streams, avoid threading. Instead: (1) put frames on the queue synchronously in a fake InputStream `__enter__`, (2) append a `None` sentinel to stop the consumer loop. File source mode (`file_source=` parameter) needs no mocking at all.
- **T010**: `LD_LIBRARY_PATH` in the nix devshell doesn't include portaudio or stdenv.cc.cc.lib — numpy and sounddevice both need these. Use `nix eval --raw nixpkgs#stdenv.cc.cc.lib`/lib and `nix eval --raw nixpkgs#portaudio`/lib.
- **T011**: `webrtcvad` fails to import in nix because it depends on `pkg_resources` (setuptools) at import time. Same pattern as sounddevice: use lazy import (`_import_webrtcvad()`) and dependency injection (`_webrtcvad_mod` / `_silero_fn` constructor kwargs) for testability. This avoids `@patch` on module-level imports entirely.
- **T011**: For components with heavy native dependencies (webrtcvad, onnxruntime), constructor-based dependency injection (`_webrtcvad_mod=`, `_silero_fn=`) is cleaner than `@patch` decorators — no import-order issues, no `AttributeError: module has no attribute` from patch targets.
- **T012**: openWakeWord expects 1280-sample chunks at 16kHz, but our audio pipeline produces 480-sample frames (30ms). The `WakeWordDetector` must accumulate input frames in a buffer and only call `model.predict()` when >= 1280 samples are available. Test `detect_on_call` values must account for this: N oww calls requires ceil(N * 1280 / 480) input frames.
- **T012**: Same dependency injection pattern as vad.py works for openWakeWord: `_oww_model=` constructor kwarg injects a mock model. Lazy import via `_import_openwakeword()` avoids loading TFLite at module import time.
- **T014**: `transcriber.py` follows the same lazy-import + dependency injection pattern (`_faster_whisper_fn` kwarg) as vad.py and wakeword.py. The `Transcriber` class converts int16 audio to float32 normalized to [-1,1] before passing to faster-whisper. Running tests outside `nix develop` requires `LD_LIBRARY_PATH` pointing to both gcc lib (libstdc++.so.6) and zlib (libz.so.1) from the nix store.
- **T015**: Pipeline uses dependency injection for VAD, transcriber, and wakeword (`_vad=`, `_transcriber=`, `_wakeword=` kwargs). After speech ends via VAD, do NOT call `vad.reset()` or `wakeword.reset()` — the VAD is already in a clean non-speech state, and resetting can cause fake test components to re-trigger on remaining silence frames. The pipeline accumulates its own `_speech_frames` during speech for max utterance duration enforcement; when VAD emits SpeechEnd, it uses the VAD's accumulated audio instead (which is authoritative).
- **T015**: For LD_LIBRARY_PATH outside nix devshell, must use 64-bit gcc lib — the 32-bit variant causes "wrong ELF class: ELFCLASS32" errors. Check ELF class byte (offset 4: 1=32-bit, 2=64-bit) to identify correct path.
- **T016**: The `SocketServer` uses asyncio `start_unix_server` + async context manager pattern. Callbacks (`on_config`, `on_control`, `on_disconnect`) are async callables set as attributes. The server sends a "ready" StatusMessage on client connect. No native deps needed — pure Python asyncio, so tests run without LD_LIBRARY_PATH.
- **T018**: The `src/protocol.ts` file and `tests/unit/ts/protocol.test.ts` were pre-scaffolded (interfaces + test shells existed but type guards and validation were missing). The `deserialize` function needs to validate both message type (reject unknown types) and required fields per type — without field validation, malformed messages silently pass through as the `Message` union type.
- **T018**: Node/npm are only available inside `nix develop`. Outside it, use the nix store path directly: `PATH="/nix/store/.../nodejs-.../bin:$PATH" node node_modules/.bin/vitest run`.
- **T017**: `__main__.py` bridges blocking audio I/O (sounddevice runs in a thread via `run_in_executor`) with the async socket server. Frames are passed from the capture thread to the asyncio event loop via `loop.call_soon_threadsafe(queue.put_nowait, frame)`. `VoiceError` exceptions from audio capture are also passed through the queue and converted to `ErrorMessage` protocol messages.
- **T020**: `src/config.ts` is straightforward — `vscode.workspace.getConfiguration("claude-voice")` plus `.get()` with defaults for each field. The config watcher uses `onDidChangeConfiguration` filtered by `affectsConfiguration("claude-voice")`. Tests pre-existed in `tests/unit/ts/config.test.ts` with a well-structured vscode mock that exposes `_configValues` for mutation and `_fireConfigChange` to simulate settings events.
- **T019**: `src/socket-client.ts` and its test were pre-scaffolded and nearly complete. The exponential backoff test failed because `vi.runAllTimersAsync()` drains ALL pending timers (including timers created by reconnect failures), causing the test to race ahead. Fix: use `vi.advanceTimersByTimeAsync(0)` after advancing to a timer fire point — this flushes microtasks without advancing the clock further. Also, `@types/node` was missing from devDependencies, causing typecheck to fail for `node:events`, `node:net`, `Buffer`, and `setTimeout` — added it.
- **T043**: Vitest custom reporters are plain ES module classes with `onInit()` and `onFinished(files)` hooks. The `files` parameter contains nested `tasks` arrays — use recursive flattening to get leaf tests. For pytest, a conftest.py at the project root with `pytest_plugins = ["tests.pytest_json_reporter"]` registers the plugin. The `pytest_runtest_logreport` hook fires per test phase (setup/call/teardown) — filter to `when == "call"` for actual test results.
- **T049**: The `src/logger.ts` uses a shared OutputChannel singleton (`vscode.window.createOutputChannel("Claude Voice")`) so all modules log to the same panel. Log level is read dynamically from `claude-voice.logLevel` setting on each log call (not cached), so changes take effect immediately without recreating loggers. The `package.json` needed a `claude-voice.logLevel` configuration property added (enum: debug/info/warn/error, default: info).
- **T047**: `ShutdownRegistry` uses `asyncio.wait_for` with `run_in_executor` for sync hooks and direct `await` for async hooks. This lets the same timeout mechanism work for both sync (blocking) and async cleanup functions. The `time.monotonic()` deadline pattern enforces the overall timeout across all hooks — once the deadline passes, remaining hooks are skipped.
- **T022**: `SidecarManager` extends `EventEmitter<SidecarManagerEvents>` for typed events (started, stopped, error). The circuit breaker uses a sliding window of crash timestamps — timestamps older than 60s are pruned on each crash. The `error` event on Node.js EventEmitter throws if no listener is attached, so tests that trigger the circuit breaker must add an `on("error", () => {})` listener. The mock `ChildProcess` for tests needs a custom `kill` that sets `killed = true` and emits `close`.
- **T046**: The Python sidecar logger uses `contextvars.ContextVar` for correlation ID propagation — this is cleaner than thread-local storage and works correctly with `asyncio`. The `with_correlation_id()` context manager uses `ContextVar.set()`/`reset()` with tokens, which correctly handles nesting (inner context restores outer value on exit). The custom `logging.Formatter` reads the context var at format time, so any handler attached to the root logger automatically gets correlation IDs without special configuration.
- **T051**: When testing `sys.excepthook` assignment, the hook may already be set from a previous test or import in the same pytest process. Reset to `sys.__excepthook__` before asserting that `main()` sets it. The `--check` flag uses lazy local imports (`from sidecar.audio import _import_sounddevice`) to avoid loading native deps at module level — consistent with the project's lazy-import pattern.
- **T023**: The `ClaudeBridge` sequential delivery queue uses a simple promise-chain pattern: each `deliver()` call chains onto `this._queue` via `.then()`, and `this._queue` is updated to point to the new tail (with `.catch(() => {})` to avoid unhandled rejections). This ensures FIFO ordering without explicit mutex or queue data structure. The `vscode.commands.executeCommand('type', { text })` API accepts arbitrary text (not just single characters), so we pass the full transcript as one `type` command rather than character-by-character. Enter is simulated as `{ text: "\n" }`.
- **T041**: CI uses `DeterminateSystems/nix-installer-action` + `magic-nix-cache-action` for Nix in GitHub Actions — much faster than `cachix/install-nix-action` due to built-in caching. All build/test steps run inside `nix develop --command bash -c "..."` for reproducibility. ruff was not in dev dependencies — added to pyproject.toml `[tool.uv] dev-dependencies` and `[project.optional-dependencies] dev`. Security scan job (Trivy, Semgrep, Gitleaks) runs independently without Nix since those tools are self-contained GitHub Actions.
